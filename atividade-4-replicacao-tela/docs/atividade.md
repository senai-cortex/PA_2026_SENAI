# ATIVIDADE EM DUPLAS

## Capacidades:

- CT7 - Reconhecer especificações técnicas e paradigmas de linguagem de programação.
- CS3 - Faz as atividades com organização.

## Objetivo:

- Replicar, com o maior rigor possível, duas páginas do SAP utilizando HTML e CSS, com foco na observação e reprodução de leiaute, organização visual, estrutura de seções, alinhamentos, espaçamentos, dimensões e padronização de estilos.

## Produto esperado:

- Entregar um repositório contendo duas páginas HTML e um arquivo CSS, desenvolvidos para reproduzir visualmente duas páginas do SAP, com organização clara dos arquivos e fidelidade ao modelo proposto.

## O repositório deve apresentar:

- [ ] duas páginas HTML correspondentes aos dois leiautes solicitados;
- [ ] um único arquivo CSS externo para estilização das páginas;
- [ ] reprodução visual das páginas com o maior rigor possível;
- [ ] estrutura organizada e padronizada entre os arquivos;
- [ ] nomes de arquivos claros e coerentes;
- [ ] organização visual que facilite a validação da atividade.

# Rubricas (Critérios de avaliação)

(CT7)
Insuficiente: o grupo não conseguiu reproduzir as páginas propostas; apresentou estrutura muito diferente do modelo; houve uso incorreto ou muito limitado de HTML e CSS; o resultado final ficou desorganizado e distante da composição visual esperada.
Básico: o grupo reproduziu parcialmente as páginas; demonstrou compreensão inicial da estrutura em HTML e da estilização em CSS, mas com inconsistências visíveis de alinhamento, espaçamento, proporção, hierarquia visual ou fidelidade ao modelo.
Adequado: o grupo reproduziu corretamente as duas páginas propostas; utilizou HTML e CSS com coerência; os elementos ficaram organizados, com boa aproximação visual em relação ao modelo solicitado.
Avançado: além do adequado, o grupo alcançou alta fidelidade visual nas duas páginas, manteve consistência entre os arquivos, organizou o CSS com clareza, evitou repetições desnecessárias e demonstrou cuidado técnico e estético na composição final.
 
(CS3)
Insuficiente: o repositório está desorganizado; faltam arquivos; a estrutura dificulta a leitura, navegação ou validação da atividade.
Básico: o repositório apresenta organização parcial; os arquivos estão presentes, mas há pouca padronização, pouca clareza na estrutura ou baixa revisão do código.
Adequado: o repositório apresenta estrutura clara, separação correta entre HTML e CSS, arquivos identificados e entrega completa.
Avançado: além do adequado, o grupo demonstra planejamento técnico, padronização consistente entre as páginas, boa legibilidade do código e cuidado na apresentação final do repositório.

# Instruções de entrega para o código:

> Crie um repositório novo no GitHub com o nome atividade_ (minúsculo, sem acentos, sem espaços, usando _). No repositório, organize os dois arquivos HTML e o arquivo CSS utilizados na atividade. Entregue o link do repositório no ambiente virtual de aprendizagem.
